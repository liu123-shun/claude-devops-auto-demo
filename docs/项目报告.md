# 图书借阅管理系统 — 项目详细报告

---

## 摘要

本项目基于 **FastAPI + MySQL + JWT** 技术栈，设计并实现了一个功能完整的**分角色图书借阅管理系统**。系统支持**管理员**和**学生**两种角色登录，涵盖图书管理、学生管理、借阅管理、评分评论、收藏、公告、数据可视化、系统配置等模块。后端采用分层架构（API→Service→DAO→Model），前端使用原生 HTML/CSS/JS 实现 **18 个页面**。系统集成了**基于协同过滤的个性化推荐算法**、**TF-IDF 文本相似度匹配**、**K-Means 用户聚类**等智能算法。部署方案包含 Docker 容器化编排和 GitHub Actions CI/CD 持续集成流水线。

**关键词**：图书管理、FastAPI、推荐算法、协同过滤、数据可视化、微服务架构

## 1. 项目背景与意义

### 1.1 项目背景

在高校图书馆管理工作中，传统的人工记录方式存在效率低、易出错、数据难以统计等问题。随着信息技术的快速发展，数字化图书管理系统已成为高校信息化建设的核心组成部分。

### 1.2 项目意义

- **管理效率提升**：自动化处理图书借还、库存管理、学生管理等工作，大幅降低人工操作成本
- **数据分析能力**：通过可视化图表和多维度统计，为图书馆运营提供数据支持
- **个性化服务**：基于协同过滤和内容匹配的推荐算法，为学生提供精准的图书推荐
- **技术实践价值**：涵盖前后端分离、数据库设计、算法实现、容器化部署等完整技术栈，具有较高的教学和实践参考价值

## 3. 系统总体设计

### 3.1 系统架构

采用**前后端分离 + 分层架构**设计：

```
┌─────────────────────────────────────────────────────────────┐
│                    前端层（18 个 HTML 页面）                 │
│   原生 HTML + CSS + JavaScript                              │
│   Chart.js 数据可视化 · CSS 变量深色模式 · Fetch API 通信   │
├─────────────────────────────────────────────────────────────┤
│                    API 路由层（FastAPI）                     │
│   auth.py（6个端点） · admin.py（30+端点） · student.py（25+端点）│
├─────────────────────────────────────────────────────────────┤
│                    业务逻辑层（Service）                     │
│   auth_service · book_service · reader_service              │
│   borrow_service（借阅限制 · 逾期拦截 · 库存变更）           │
├─────────────────────────────────────────────────────────────┤
│                    数据访问层（DAO）                         │
│   10 个 DAO 模块 · SQLAlchemy ORM · 分页查询 · 聚合统计      │
├─────────────────────────────────────────────────────────────┤
│                    MySQL 8.0 数据库                          │
│   10 张数据表 · 2 个触发器 · 自动建表 · 自迁移 Schema        │
│   Docker Volume 数据持久化                                   │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 技术选型

| 层级 | 技术 | 选择理由 |
|------|------|----------|
| **后端框架** | FastAPI | 高性能异步框架，自动生成 Swagger 文档，类型安全 |
| **ORM** | SQLAlchemy 2.0 | 成熟稳定，支持复杂查询和关联映射 |
| **数据库** | MySQL 8.0 | 功能完善，支持触发器、视图，生产环境首选 |

| **前端** | 原生 HTML/CSS/JS | 无框架依赖，部署简单，文件体积小 |

## 5. 详细功能实现

### 5.1 鉴权模块（auth.py）

#### 5.1.1 登录流程

```
用户输入 → 获取图形验证码 → 输入账号+密码+验证码+选择角色
→ POST /api/auth/login → 校验验证码有效性
→ 校验账号和密码（bcrypt.verify）→ 写入 login_log
→ 签发 JWT Token（2小时）→ 返回 Token + 用户信息
```

**核心代码逻辑**：
- 验证码通过 JWT 加密存储（5 分钟有效），防止客户端篡改
- 密码使用 `bcrypt.hashpw()` 加密存储，`bcrypt.checkpw()` 校验
- JWT payload 包含 `user_id`、`username`、`role`、`exp`

#### 5.1.2 验证码生成

使用 Python Pillow 库生成 4 位字母数字验证码图片：
- 随机生成 4 位字符（大写字母+数字）
- 背景噪声线 + 随机色彩文字 + 干扰点
- 输出为 PNG 格式的 Base64 Data URI
- 验证码值通过 JWT 加密返回，5 分钟有效期

#### 5.1.3 权限拦截实现

```python
# 从 HTTPBearer Header 解析 Token
def get_current_user(credentials):
    payload = decode_jwt_token(credentials)
    if payload is None: raise HTTPException(401)  # 未登录
    
# 角色拦截
def require_admin(current_user):
    if role != "admin": raise HTTPException(403)   # 无权限
```

#### 5.1.4 自助注册

学生注册流程：
```
填写信息（账号/姓名/密码/班级/手机号+验证码）
→ 校验验证码 → 校验用户名唯一性
→ 创建 sys_user（bcrypt加密密码）→ 创建 reader（绑定账号）
→ 自动签发 JWT Token → 跳转学生首页
```

### 5.2 图书管理模块

#### 5.2.1 图书 CRUD

| 操作 | 接口 | 实现 |
|------|------|------|
| 新增 | POST `/api/admin/books` | 校验必填字段 → 自动根据 ISBN 获取封面 → 插入 book 表 |
| 列表 | GET `/api/admin/books` | 支持 keyword 搜索（书名/作者/ISBN） + category 分类筛选 + 分页 |
| 详情 | GET `/api/admin/books/{id}` | 返回图书全部字段 + 最近借阅记录 + 评分数据 + 最新评论 |
| 修改 | PUT `/api/admin/books/{id}` | 部分更新（Pydantic exclude_none），仅更新传入字段 |
| 删除 | DELETE `/api/admin/books/{id}` | 删除图书记录 |

#### 5.2.2 封面自动抓取

```
新增图书时 → 检查 cover_url 是否为空 → 若为空且有 ISBN
→ 调用 OpenLibrary API: https://covers.openlibrary.org/b/isbn/{isbn}-L.jpg
→ HEAD 请求验证封面存在 → 存入 cover_url 字段
```

#### 5.2.3 批量导入

上传 CSV 文件 → 自动识别 UTF-8/GBK 编码 → DictReader 逐行解析 → 创建 Book 对象 → 批量提交

支持中英文列名（如 `book_name` 和 `书名` 均可识别）。

### 5.3 借阅管理模块

#### 5.3.1 借书流程

```
学生选择图书 → POST /api/student/borrows {book_id, reader_id}
→ Service 层校验：
   ① 图书是否存在、库存是否 > 0
   ② 当前学生是否超过最大借书数（从 system_config 读取 max_borrow_count）
   ③ 是否有逾期未还的图书
→ 校验通过 → 创建 borrow_record → 触发器自动扣库存 + 写操作日志
→ book.total_borrows + 1
```

#### 5.3.2 还书流程

```
学生点击还书 → PUT /api/student/borrows/{id}/return
→ 校验：该借阅记录属于当前学生本人
→ 校验：尚未归还
→ 更新 return_time = now → 触发器自动加库存 + 写归还日志
```

#### 5.3.3 逾期处理

```
逾期判定：return_time 为空 且 borrow_time < (now - overdue_days)
逾期天数阈值从 system_config 表 overdue_days 读取（默认30天）
逾期页面支持关键词搜索 + 直接还书操作
```

### 5.4 评分评论模块

#### 5.4.1 评分规则

- 仅归还过该书的学生才可评分
- 每本书每人只能评分一次
- 评分范围 1-5 星
- 评论内容可选，最多 1000 字符

#### 5.4.2 评论管理

- 学生可修改/删除自己的评论（PUT/DELETE `/api/student/reviews/{id}`）
- 管理员可删除不当评论
- 评论编辑时复用评分星星选择器

#### 5.4.3 图书详情评分展示

```
图书详情 = 基础信息 + avg_rating（平均分，保留1位小数）
         + review_count（评价总数）
         + reviews[]（最新5条评论，包含评分/内容/用户/时间）
```

## 6. 推荐算法实现

### 6.1 算法架构

系统共实现 **5 种推荐策略**，通过 Tab 按钮切换：

| 序号 | 算法名称 | 算法类型 | 核心逻辑 |
|------|----------|----------|----------|
| 1 | 猜你喜欢 | 分类协同 | 基于用户借阅历史的分类偏好，推荐同类热门书 |
| 2 | 高分推荐 | 评分排序 | 全站评分 ≥ 4 星的图书，按平均评分降序 |
| 3 | 全站热门 | 统计排名 | 全站借阅次数最多的 Top N 图书 |
| 4 | 冷门好书 | 逆热度推荐 | 评分高（≥4星）但借阅次数少（<5次）的宝藏书 |
| 5 | 个性化推荐 | 协同过滤 | 基于用户相似度的协同过滤算法 |

### 6.2 协同过滤算法实现（核心）

#### 6.2.1 算法流程

```
输入：当前用户 user_A
Step 1 — 相似用户发现：
    查询 user_A 借过的所有图书 book_list_A
    查找也借过 book_list_A 中任意一本的其他用户 → similar_users[]

Step 2 — 关联书籍统计：
    对 similar_users 中的每个用户，收集他们借过的其他书
    统计每本书被多少个相似用户借过 → book_count[]

Step 3 — 排除已有：
    排除 user_A 已经借过的书

Step 4 — 排序输出：
    按 book_count 降序排列，取 Top N 返回

Step 5 — 冷启动处理：
    如果结果 < 3 本（新用户或数据稀疏），自动退化到热门推荐
```

#### 6.2.2 算法特点

- **用户相似度**：使用 Jaccard 相似度思想，共同借阅的图书越多，认为用户越相似
- **冷启动处理**：新用户无借阅历史时，自动退化为热门推荐
- **数据补偿**：协同过滤结果不足时，用同分类热门图书补充
- **实时计算**：推荐结果基于最新数据库实时查询，无需离线训练
